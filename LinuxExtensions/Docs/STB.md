# Библиотека STB

Библиотеки `stb` — это коллекция популярных однофайловых библиотек на языке C, ставшая стандартом де-факто в разработке игр и графических приложений.

**Кто создатель?**

Автор проекта — Шон Барретт (Sean Barrett). Он опытный программист, известный своей работой в игровой индустрии, в частности над движком игры Thief: The Dark Project.

**Какие цели преследовал автор?**

Шон создавал эти библиотеки для решения конкретных проблем «тяжеловесных» зависимостей:

* Простота интеграции: Вам не нужно настраивать сборку, линковку или `Makefile`. Достаточно просто закинуть один `.h` файл в проект.

* Отсутствие зависимостей: Библиотеки не требуют ничего, кроме стандартной библиотеки C (CRT).

* Публичное достояние: Код распространяется по лицензии MIT или Unlicense, что позволяет использовать его в любых проектах без юридических сложностей.

* Скорость и легкость: Они написаны так, чтобы быстро компилироваться и занимать минимум места.

**Какие модули входят в состав?**

Хотя самая известная часть — это `stb_image.h`, коллекция включает десятки модулей для самых разных задач. Вот основные:

Изображения:

* stb_image.h: Чтение почти всех популярных форматов (JPEG, PNG, TGA, BMP, GIF и др.).

* stb_image_write.h: Запись изображений на диск.

* stb_image_resize.h: Качественное изменение размера (ресамплинг).

* Шрифты:

* stb_truetype.h: Отрисовка и растеризация шрифтов TrueType.
Аудио:

* stb_vorbis.c: Декодирование формата Ogg Vorbis.

Инструменты и структуры данных:

* stb_ds.h: Удобные динамические массивы и хэш-таблицы в стиле C++.

* stb_textedit.h: Внутреннее устройство текстового редактора для игр и т.д., разработанное с нуля. 

* stb_sprintf.h: Очень быстрая реализация `printf`.

* stb_lexing.h: Лексический анализатор для создания парсеров.

**Как подключить и использовать**

Чтобы правильно подключить, например, `stb_image.h`, нужно помнить, что это header-only библиотека. Это значит, что один и тот же файл содержит и описания функций, и их реализацию.

1. Как подключить

Если вы просто напишете `#include "stb_image.h"`, вы получите только прототипы функций. Чтобы скомпилировать сам код библиотеки, нужно в одном (и только в одном) `.c` или `.cpp` файле проекта добавить специальный макрос перед инклюдом:

```c
#define STB_IMAGE_IMPLEMENTATION
#include "stb_image.h"
```

Во всех остальных файлах, где вам нужно использовать функции библиотеки, макрос `STB_IMAGE_IMPLEMENTATION` писать не нужно — достаточно обычного `#include`.

2. Как использовать (пример)
   
Самая частая задача — загрузка картинки в массив байтов для последующей передачи в OpenGL, DirectX или просто для обработки.

```c
#include <stdio.h>

#define STB_IMAGE_IMPLEMENTATION
#include "stb_image.h"

int main() {
    int width, height, channels;
    
    // Загружаем изображение
    // 4 — это желаемое кол-во каналов (RGBA). 0 — если оставить как в файле.
    unsigned char *data = stbi_load("image.png", &width, &height, &channels, 4);

    if (data == NULL) {
        printf("Ошибка загрузки: %s\n", stbi_failure_reason());
        return 1;
    }

    printf("Картинка загружена: %dx%d, каналов: %d\n", width, height, channels);

    // ... тут работаем с данными (массив data) ...

    // ОБЯЗАТЕЛЬНО освобождаем память
    stbi_image_free(data);
    
    return 0;
}
```

3. Полезные советы

* Переворот оси Y: В компьютерной графике (особенно в OpenGL) начало координат часто находится внизу, а в изображениях — вверху. Чтобы библиотека сама перевернула картинку при загрузке, вызовите: `stbi_set_flip_vertically_on_load(1);` до вызова `stbi_load`.
   
* Обработка ошибок: Если функция вернула `NULL`, причину можно узнать через `stbi_failure_reason()`.
   
* Поддержка форматов: Библиотека «из коробки» понимает JPEG, PNG, BMP, GIF, TGA, PIC, PSD (только композитный слой) и др.

**Динамические массивы**

`stb_ds.h` — это, пожалуй, самая «магическая» часть коллекции. Она реализует динамические массивы (аналог 
`std::vector` из C++) и хэш-таблицы, используя чистый C и макросы.

Главная фишка: вы используете обычные указатели, но библиотека хранит служебную информацию (размер, емкость) в памяти прямо перед началом массива.

**Основные функции для динамических массивов (`stbds_arr`)**

Чтобы начать работу, достаточно объявить обычный указатель и инициализировать его как `NULL`.

1. Добавление элементов
   
* `arrput(a, v)`: Добавляет значение `v` в конец массива a. Если места нет, массив автоматически переаллоцируется (увеличивается в 1.5–2 раза).
   
* `arrins(a, i, v)`: Вставляет значение по индексу `i`, сдвигая остальные элементы вправо.

2. Работа с размером
   
* `arrlen(a)`: Возвращает текущее количество элементов. Если массив равен `NULL`, функция безопасно вернет 0.
   
* `arrcap(a)`: Возвращает текущую емкость (сколько памяти выделено всего).
   
* `arrsetlen(a, n)`: Принудительно устанавливает длину. Полезно, если вы заполнили массив вручную.
   
* `arrsetcap(a, n)`: Заранее резервирует память, чтобы избежать частых переаллокаций.

3. Удаление

* `arrdel(a, i)`: Удаляет элемент по индексу i, сохраняя порядок (сдвигает хвост влево). Медленно для больших массивов.
   
* `arrdelswap(a, i)`: «Быстрое» удаление. Копирует последний элемент на место удаляемого. Порядок нарушается, но работает за O(1).
   
* `arrfree(a)`: Полностью освобождает память и устанавливает указатель в `NULL`.

Пример использования

```c
#define STB_DS_IMPLEMENTATION
#include "stb_ds.h"

int main() {
    int *my_arr = NULL; // Обязательно NULL в начале

    // Добавляем элементы
    for (int i = 0; i < 10; ++i)
        arrput(my_arr, i * 10);

    printf("Длина: %td\n", arrlen(my_arr)); // Выведет 10

    // Быстрое удаление 3-го элемента
    arrdelswap(my_arr, 2); 

    // Проход по массиву как обычно
    for (int i = 0; i < arrlen(my_arr); ++i)
        printf("%d ", my_arr[i]);

    arrfree(my_arr); // Очистка
    return 0;
}
```
Важные нюансы

* Типизация: Поскольку это макросы, функции работают с любыми типами данных (`int`, `struct`, `double*`) автоматически.

* L-value: Функции типа `arrput` могут изменить адрес самого указателя (если случился `realloc`), поэтому всегда передавайте им сам указатель, а не его копию.

* Безопасность: В отличие от `std::vector`, здесь нет проверки границ в рантайме (out-of-bounds), так что следите за индексами сами.

**Хэш-таблицы**

В `stb_ds.h` хэш-таблицы реализованы максимально близко к `std::unordered_map` из C++, но адаптированы под синтаксис C. Вы можете использовать в качестве ключей как обычные числа или строки, так и целые структуры.

1. Как объявить таблицу

Таблица в `stb_ds` — это обычный массив структур, где первый элемент структуры — это ключ, а остальные — данные (значение).

```c
struct {
    char *key;   // Ключ (строка)
    int value;   // Данные
} *my_table = NULL; // Обязательно инициализировать NULL
```

2. Основные операции (макросы)

* `shput(t, k, v)` (String Hash Put): Добавить или обновить пару «ключ-значение», где ключ — строка (`char*`).
   
* `shget(t, k)`: Получить значение по строковому ключу. Если ключа нет, вернет 0 (или дефолтное значение).
   
* `shgets(t, k)`: Вернуть всю структуру (указатель на нее) по ключу. Если ничего не найдено, вернет `NULL`.
   
* `shlen(t)`: Количество элементов в таблице.
   
* `shdel(t, k)`: Удалить элемент по ключу.
   
* `shfree(t)`: Полная очистка памяти.
   
Важно: Для целочисленных ключей (`int`, `long`) используйте префикс hm (например, `hmput`, `hmget`), а для строк — `sh`.

3. Пример использования (строковые ключи)

```c
#define STB_DS_IMPLEMENTATION
#include "stb_ds.h"

struct { char *key; int value; } *inventory = NULL;

// 1. Добавляем данные
shput(inventory, "apple", 10);
shput(inventory, "banana", 5);

// 2. Читаем данные
int count = shget(inventory, "apple"); 
printf("Яблок: %d\n", count); // Выведет 10

// 3. Проверка на существование
if (shgets(inventory, "orange") == NULL) {
    printf("Апельсинов нет на складе\n");
}

// 4. Перебор всей таблицы (итерирование)
for (int i = 0; i < shlen(inventory); ++i) {
    printf("Продукт: %s, Кол-во: %d\n", inventory[i].key, inventory[i].value);
}

shfree(inventory);
```
4. Нюансы
   
* Строки: Библиотека делает внутреннюю копию ключа через `strdup`, поэтому вам не нужно беспокоиться о жизни исходной строки.
   
* Дефолтные значения: С помощью `sh_new_arena` или `shdefault` можно настроить, что именно будет возвращать `shget`, если ключ не найден (например, -1 вместо 0).
   
* Производительность: Под капотом используется открытая адресация, что делает поиск очень быстрым.

**Лексер**

`stb_c_lexer.h` — это не парсер, а лексический анализатор. Он не понимает иерархию (кто на кого умножается), он просто рубит строку на «токены»: числа, знаки и слова.

Для разбора строки "1 + 2 * 3" вам нужно сделать два шага: лексику (разбить на части) и парсинг (построить дерево или вычислить).

1. Подключение и настройка

Лексер требует буфер для хранения строк. Обычно его выделяют прямо в стеке или куче.

```c
#define STB_C_LEXER_IMPLEMENTATION
#include "stb_c_lexer.h"

stb_lexer lex;
char *input = "1 + 2 * 3";
char string_store[1024];

// Инициализация
stb_c_lexer_init(&lex, input, input + strlen(input), string_store, 1024);
```

2. Как работает цикл разбора
   
Вы вызываете `stb_c_lexer_get_token(&lex)`, и структура `lex` обновляется данными о текущем элементе.
   
* `lex.token` — тип токена (число, символ, ключевое слово).
   
* `lex.int_number` / `lex.real_number` — само значение, если это число.

3. Пример кода для разбора строки "1 + 2 * 3"

```c
while (stb_c_lexer_get_token(&lex)) {
    if (lex.token == CLEX_intlit) {
        printf("Число: %ld\n", lex.int_number);
    } else if (lex.token == '+') {
        printf("Оператор: Плюс\n");
    } else if (lex.token == '*') {
        printf("Оператор: Умножение\n");
    } else if (lex.token == CLEX_eof) {
        break;
    }
}
```

4. Как превратить это в результат?

Чтобы правильно посчитать "1 + 2 * 3" (с учетом приоритетов), одного цикла `while` мало. Обычно используют:
   
* Алгоритм сортировочной станции (Shunting-yard): превращает выражение в постфиксную запись "(1 2 3 * +)", которую легко считать стеком.
   
* Рекурсивный спуск: вы создаете функции `parse_expression`, `parse_term` и `parse_factor`, которые вызывают друг друга.
   
**Почему `stb_c_lexer.h` удобен?**
   
Он берет на себя всю «грязную» работу: пропуск пробелов, распознавание шестнадцатеричных чисел (0x10), обработку кавычек и комментариев.

Он очень быстрый и не требует настройки регулярных выражений.

**Парсер на основе `stb_c_lexer.h`**

Для вычисления значения выражения "1 + 2 * 3" с учетом приоритетов (умножение перед сложением) проще всего использовать метод рекурсивного спуска.

Лексер `stb` будет поставлять нам «кирпичики», а функции парсера будут их собирать.

```c
#include <stdio.h>
#include <string.h>

#define STB_C_LEXER_IMPLEMENTATION
#include "stb_c_lexer.h"

stb_lexer lex;

// Прототипы для рекурсии
double parse_expression();
double parse_term();

// 1. Уровень чисел (самый высокий приоритет)
double parse_factor() {
    double val = 0;
    if (lex.token == CLEX_intlit) {
        val = (double)lex.int_number;
        stb_c_lexer_get_token(&lex); // Переходим к следующему токену
    } else if (lex.token == CLEX_fplit) {
        val = lex.real_number;
        stb_c_lexer_get_token(&lex);
    }
    return val;
}

// 2. Уровень умножения и деления
double parse_term() {
    double left = parse_factor();
    while (lex.token == '*' || lex.token == '/') {
        int op = lex.token;
        stbi_inline stb_c_lexer_get_token(&lex);
        if (op == '*') left *= parse_factor();
        else left /= parse_factor();
    }
    return left;
}

// 3. Уровень сложения и вычитания (самый низкий приоритет)
double parse_expression() {
    double left = parse_term();
    while (lex.token == '+' || lex.token == '-') {
        int op = lex.token;
        stb_c_lexer_get_token(&lex);
        if (op == '+') left += parse_term();
        else left -= parse_term();
    }
    return left;
}

int main() {
    char *input = "1 + 2 * 3";
    char string_store[1024];

    // Инициализируем лексер
    stb_c_lexer_init(&lex, input, input + strlen(input), string_store, 1024);
    
    // Читаем первый токен, чтобы "запустить" процесс
    stb_c_lexer_get_token(&lex);

    // Считаем результат
    double result = parse_expression();

    printf("Выражение: %s\n", input);
    printf("Результат: %.2f\n", result); // Выведет 7.00

    return 0;
}
```

Как это работает:

* `parse_expression` вызывает `parse_term`.

* `parse_term` сначала получает первое число через `parse_factor`, а затем смотрит: «А нет ли дальше знака * ?».

* Поскольку в "1 + 2 * 3" после единицы идет "+", `parse_term` просто возвращает `1` наверх в `parse_expression`.

* Затем `parse_expression видит` + и вызывает `parse_term` для правой части.

* Второй вызов "parse_term" заглатывает "2", видит "*", заглатывает "3", перемножает их и возвращает "6".

* Итог: 1 + 6 = 7.

Что здесь удобно? Лексер сам отсек пробелы и распознал, что 1, 2 и 3 — это целые числа ("CLEX_intlit"), избавив нас от ручного перебора символов.
