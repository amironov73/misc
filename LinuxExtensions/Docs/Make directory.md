# Создание директории

Функция `mkdir` описана в заголовочном файле `<sys/stat.h>`. В некоторых системах (согласно стандартам POSIX) также может потребоваться подключение `<sys/types.h>`.

## О коде mkdir("workdir/iogunb", 0755);

В стандартном исполнении эта функция не поймет такой вызов, если директория `workdir` еще не создана.
Вот ключевые моменты:

1. Отсутствие рекурсии: Стандартная функция `mkdir` в C умеет создавать только одну (последнюю в пути) директорию за раз. Если папки `workdir` не существует, вызов вернет `-1`, а переменная errno примет значение `ENOENT` ("No such file or directory").

2. Решение: Чтобы этот вызов сработал, вам нужно сначала создать `workdir`, а вторым вызовом — `iogunb`.

3. Права доступа (0755): Здесь всё верно. 0755 — это восьмеричное число, которое означает:

    * 7 (rwx) — владельцу можно всё.
    * 5 (r-x) — группе и остальным можно читать и заходить в папку.
    * Важно: Реальные права будут еще зависеть от `umask` вашего процесса (обычно итоговые права становятся чуть строже).

## Как сделать правильно (создание вложенных папок)

Если вам нужно поведение как у команды `mkdir -p` (создать всю цепочку), придется написать небольшую вспомогательную функцию:

```c++
#include <sys/stat.h>
#include <sys/types.h>
#include <errno.h>

// Пример простого создания по очереди
void setup_dirs() {
    mkdir ("workdir", 0755);
    if (mkdir ("workdir/iogunb", 0755) == 0) {
        // Успех
    }
}
```

Как сработает `mkdir`, если нужная директория уже существует? Если директория уже существует, функция `mkdir` завершится с ошибкой. Она не проверяет права доступа и не пытается «обновить» существующую директорию. Если на существующую директорию установлены другие права, вы об этом не узнаете. Логика `mkdir` проста: если имя занято, функция возвращает сигнал о неудаче. Вот что именно произойдет:

1. Возвращаемое значение: Функция вернет -1.

2. Код ошибки (`errno`): Глобальная переменная errno будет установлена в значение `EEXIST` («File exists»). Это касается и случаев, когда по указанному пути существует файл с таким же именем.

3. Состояние папки: С самой директорией и её правами ничего не случится — они останутся прежними.

## Как правильно обрабатывать эту ситуацию

В программировании на C считается хорошей практикой проверять, не вызвана ли ошибка тем, что папка уже на месте. Если папка есть — для нас это часто равносильно успеху.

```c++
#include <sys/stat.h>
#include <errno.h>
#include <stdio.h>

void create_dir() {
    if (mkdir ("workdir", 0755) == -1) {
        if (errno == EEXIST) {
            // Это не проблема: папка уже была создана ранее
            printf ("Директория уже существует, продолжаем работу.\n");
        } else {
            // А это реальная ошибка (например, нет прав или путь неверный)
            perror ("Ошибка при создании директории");
        }
    } else {
        printf ("Директория успешно создана.\n");
    }
}
```

## Важный нюанс с правами доступа

Если папка существует, но у неё установлены другие права (например, `0700` вместо нужных вам `0755`), `mkdir` всё равно просто вернет `EEXIST`. Она не изменит права доступа на те, что вы указали в аргументах. Если вам критически важно, чтобы права были именно `0755`, после (или вместо) `mkdir` нужно использовать функцию `chmod`:

```c++
chmod ("workdir", 0755);
```
Проверить, что с директорией все хорошо, можно с помощью функции `stat()`. Функция `stat()` возвращает структуру с информацией о владельце, типе объекта и правах доступа. Однако есть важный нюанс: проверка прав через `stat()` вручную — это работа с битовыми масками, что может быть избыточным, если вам нужно просто узнать «можно ли мне туда писать».

Вот как это сделать правильно.

## 1. Проверка существования и типа (директория или нет)

Сначала проверим, что объект существует и что это именно директория, а не файл.

```c++
#include <sys/stat.h>
#include <stdio.h>

struct stat st;
if (stat ("workdir", &st) == 0) {
    if (S_ISDIR(st.st_mode)) {
        printf ("Это директория.\n");
    } else {
        printf ("Это файл, а не директория!\n");
    }
} else {
    perror ("Объект не найден");
}
```

## 2. Проверка прав на запись

Хотя `stat` содержит биты прав (`st.st_mode`), проверять их вручную сложно (нужно учитывать, являетесь ли вы владельцем, входите ли в группу и т.д.). Для проверки возможности записи в Linux лучше использовать функцию `access()` из `<unistd.h>`. Она учитывает реальный ID пользователя и все нюансы прав.

```c++
#include <unistd.h>

if (access ("workdir", W_OK) == 0) {
    printf ("Запись разрешена.\n");
} else {
    perror ("Доступ на запись запрещен");
}
```

## 3. Комбинированный подход (Всё вместе)

Вот готовый фрагмент кода, который делает именно то, что нужно:

```c++
#include <stdio.h>
#include <sys/stat.h>
#include <unistd.h>
#include <errno.h>

int check_dir_writable (const char *path) {
    struct stat st;

    // 1. Проверяем существование и тип через stat
    if (stat (path, &st) != 0) {
        if (errno == ENOENT) printf ("Путь не существует.\n");
        return 0; 
    }

    if (!S_ISDIR (st.st_mode)) {
        printf ("Путь существует, но это не директория.\n");
        return 0;
    }

    // 2. Проверяем права на запись (W_OK) и выполнение (X_OK)
    // Для директорий X_OK нужен, чтобы иметь возможность "войти" в них
    if (access (path, W_OK | X_OK) == 0) {
        return 1; // Всё отлично, писать можно
    }

    return 0; // Существует, но писать нельзя
}
```

## Почему `access()` лучше, чем ручной разбор `st_mode` из `stat()`?

* `stat()` просто говорит вам, какие биты установлены на файле.

* `access()` учитывает, кто запустил программу. Если папку создал `root` с правами `0700`, а вы запустили программу от обычного пользователя, `stat` покажет наличие прав владельца, но `access` честно скажет: «вам сюда нельзя».

Важный момент: Если ваша папка находится на расшаренном сетевом ресурсе, смонтированным с флагом `ro` (read-only), то `access(..., W_OK)` вернет ошибку, так как сама файловая система запрещает запись, даже если биты прав на папке говорят об обратном.
