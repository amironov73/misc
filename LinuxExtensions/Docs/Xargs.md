# Xargs

Команда xargs (от англ. extended arguments) — это мощная утилита в Linux и Unix-подобных системах, которая преобразует данные из стандартного ввода (stdin) в аргументы для других команд.

## Зачем она нужна?

Многие команды (например, `rm`, `cp`, `mkdir`) не умеют читать список файлов напрямую из конвейера (`|`). Они ожидают их как список аргументов после самой команды. `xargs` служит «переходником»: он берет текст, который выдала предыдущая программа, и подставляет его в конец следующей.

## Основные сценарии использования

1. Связка с поиском (`find`)

Самый частый пример — удаление старых логов или поиск файлов по шаблону:

```bash
find . -name "*.log" | xargs rm
```

Здесь `find` находит файлы, а `xargs` передает их список команде `rm`.

2. Обход ограничения на длину строки

В Linux есть лимит на количество аргументов одной команды. Если вы попытаетесь удалить миллион файлов через `rm *`, возникнет ошибка «Argument list too long». `xargs` автоматически разбивает длинный список на части и запускает команду несколько раз с оптимальным количеством аргументов.

3. Параллельное выполнение (`-P`)

Это одна из самых полезных современных функций GNU-версии `xargs`. Она позволяет запускать процессы одновременно, используя все ядра процессора:

```bash
ls *.png | xargs -P 4 -I {} convert {} {}.jpg
```

Запустит до 4 процессов конвертации изображений одновременно.

4. Удаление лишних пробелов

Если запустить `xargs` без параметров, он просто выведет текст в одну строку, убрав лишние переносы и пробелы:

```bash
echo -e "один \n два \n три" | xargs\
# Результат: один два три
```

## Полезные флаги

* `-0` (или `--null`): крайне важен при работе с именами файлов, содержащими пробелы. Используется вместе с `find -print0`.

* `-n [число]`: ограничивает количество аргументов для одного запуска команды.

* `-I {}`: позволяет вставить аргумент в середину команды, а не в конец (заменяет `{}` на текст из ввода).

* `-p`: запрашивает подтверждение перед выполнением каждой команды (полезно для безопасности).
