# Монтирование

В WSL2 монтирование сетевых ресурсов Windows (SMB/CIFS) делается достаточно просто, так как ядро WSL поддерживает файловую систему `drvfs`. Вот пошаговая инструкция:

## 1. Создайте точку монтирования

Сначала создайте папку внутри WSL, куда будет «подключен» сетевой ресурс. Обычно это делают в директории `/mnt`.

```bash
sudo mkdir -p /mnt/webshare
```

## 2. Выполните команду монтирования

Используйте тип файловой системы `drvfs`. Вам нужно будет указать ваши данные для доступа к Windows-шаре.

```bash
sudo mount -t drvfs '\\192.168.3.2\webshare' /mnt/webshare
```

Важные нюансы:

* Одинарные кавычки: Обязательно используйте одинарные кавычки '...', чтобы Linux не пытался интерпретировать обратные слэши `\` как спецсимволы.

* Права доступа: Если вы хотите, чтобы файлы принадлежали вашему пользователю Linux (а не root), добавьте параметры `-o metadata,uid=$(id -u),gid=$(id -g)`:

```bash
sudo mount -t drvfs -o "metadata,uid=$(id -u),gid=$(id -g)" '\\192.168.3.2\webshare' /mnt/webshare
```

## Если требуется логин и пароль
Если при обращении к `\\192.168.3.2\webshare` Windows запрашивает пароль, то WSL через `drvfs` может выдать ошибку. В таком случае лучше использовать стандартный протокол CIFS.

1. Установите необходимые утилиты (если еще нет):

```bash
sudo apt update && sudo apt install cifs-utils
```

2. Смонтируйте через CIFS (используя прямые слэши):

```
sudo mount -t cifs //192.168.3.2/webshare /mnt/webshare -o username=ВАШ_ЛОГИН,password=ВАШ_ПАРОЛЬ
```

## Как сделать это постоянным (автомонтирование)

Чтобы папка была доступна всегда после перезапуска WSL, добавьте строку в файл /etc/fstab:

1. Откройте файл: `sudo nano /etc/fstab`

2. Добавьте в конец (для метода `drvfs`):

```
\\192.168.3.2\webshare  /mnt/webshare  drvfs  defaults,uid=1000,gid=1000  0 0
```

(Замените 1000 на ваш ID пользователя, его можно узнать командой `id`).

Маленькая хитрость: Если вы уже подключили эту папку в самой Windows как сетевой диск (например, под буквой `Z:`), то в WSL она может появиться автоматически по пути `/mnt/z`. Проверьте это командой `ls /mnt`.
